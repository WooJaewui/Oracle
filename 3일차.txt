 # 데이터 조작하기 1 ( INSERT, SELECT )

설명.
DML : INSERT, SELECT, UPDATE, DELETE 4가지를 공부한다.

INSERT 명령규칙. ( CREATE )
INSERT INTO MEMBER(COLUMN1,COLUMN2) VALUES('data1','data2');
data를 넣지 않으면 NULL이 들어간다.

SELECT 명령규칙. ( READ )
SELECT * FROM MEMBER;
SELECT COLUMN1 FROM MEMBER;
SELECT ID (AS) NAME FROM MEMBER;

용어.
COLUMN = FIELD = 속성 : TABLE에 열을 의미한다.

주의사항.
data는 대소문자를 구분하기 때문에 입력할때 주의해야된다.
SELECT ID (AS) NAME FROM MEMBER;를 통해 ID로 저장된 DATA를 FROM이란 이름으로 READ할 수 있다.
AS는 생략이 가능하다.
빈 공백을 표현해야된다면 " "로 표현해야 된다.


 # 데이터 조작하기 2 ( UPDATE, DELETE )

UPDATE 명령규칙. ( UPDATE )
UPDATE TABLE SET COLUMN='111'; 
UPDATE TABLE SET COLUMN1='111' WHERE COLUMN2=VALUE;      
UPDATE TABLE SET COLUMN1='111', COLUMN2='222' WHERE COLUMN3=VALUE;

 # 트랜잭션 처리를 위한 COMMIT과 ROLLBACK.

용어.
트랜잭션 : 1. 업무 실행단위 - ex ) 계좌이체를 해라.
             2. 물리적인 명령단위 - ex ) 2개의 계좌에 값을 update. 
                                              ( 둘 중 하나만 실행되서는 안된다 - 트랜잭션이 깨지면 안된다 )

원리.
1. 현재 들어온 업무를 실행하기 위해 첫번째 명령어를 테스트 한다.
2. 다른 명령어들도 테스트를 진행하는 동안 다른 세션이 건드리지 못하도록 LOCK을 건다.
3. 테스트 끝나고 UNLOCK을 한다.

저장원리.
PDB에서 COMMIT하기전까지 실행한 명령어들을 저장해두었다가
COMMIT하면 모든 세션이 볼 수 있는 곳에 저장한다.

명령어.
COMMIT;     내가 지금까지 실행했던 것들을 다른세션에서도 사용 할 수 있도록 저장한다.
ROLLBACK;   내가 지금까지 실행했던 명령어들이 실행되지 않았던것처럼 되돌아간다.

주의사항.
COMMIT하지 않고 테스트를 진행중인 명령어를 다른 세션에서 실행하면 LOCK이 걸린다.



 # 연산을 통한 데이터 조회 ( 산술 연산자 )

산술연산자.
숫자 : +, -, *, /
문자 : ||

사용.
SELECT COLUMN+1 FROM NOTICE; -> SELECT COLUMN+1 COLUMN FROM TABLE;
SELECT 1+'3' FROM DUAL;  -> 4가 나온다.
SELECT 1 ||'3' FROM DUAL; -> '13'가 나온다.
SELECT 1+'A' FROM DUAL; -> 에러가 발생.

주의사항.
COLUMN SELECT할때 COLUMN + 1하면 COLUMN+1로 나옴으로 AS COLUMN을 사용해준다.
DUAL은 가상의 더미 테이블로써 연산을 해볼때 사용한다.



 # 비교연산자.

비교연산자.
=, !=, ^=, <>, IS NULL, IS NOT NULL 등.

같지않다 연산자.
!=, ^=, <> : !=와 <>를 많이 사용한다. ( 다른 언어에서 많이 쓰는 방식 몇가지를 추가함 )

사용.
SELECT * FROM TABLE WHERE COLUMN>5;
SELECT * FROM TABLE WHERE CONTENT IS NULL;
SELECT * FROM TABLE WHERE CONTENT IS NOT NULL;


 # 관계연산자.

관계연산자.
NOT, AND, OR, BETWEEN, IN.

사용.
SELECT * FROM TABLE WHERE COLUMN=0 or COLUMN=1 or COLUMN=2; ( 연속되지 않은 경우 )
SELECT * FROM TABLE WHERE COLUMN IN (0,2,7); ( 연속되지 않은 경우 )
SELECT * FROM TABLE WHERE COLUMN NOT IN (0,2,7); 
SELECT * FROM TABLE WHERE 0<=HIT AND HIT<=2; ( 연속된 경우 - 이상 미만 표현 가능 )
SELECT * FROM TABLE WHERE COLUMN BETWEEN 0 AND 2; ( 연속된 경우 - 이상 이하만 표현 가능.)
